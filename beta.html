Update

The next iteration of 2100 is based on continued refinement of the idea as well as feedback from the 2100 Devcon Alpha. It simplifies, generalizes and decentralizes the protocol, and gives 2100 general purpose utility that extends beyond the Twitter platform.

In the previous version of 2100, the relationship between 2100 tokens and Twitter handles was enforced by an oracle. A centralized oracle monitored the state of Twitter and enforced the creation, mutation or deletion of a 2100 token based on the state of its corresponding Twitter handle. The next iteration of 2100 does away with the idea of oracles completely, instead relying on a strings’ 'referent’ to create a schelling point for scarcity and value.

To understand what this means and why it might be a good idea, let’s look at an example - the system of .com domains.

If you were told that you could own one of the following three .com domains, which would you choose?

  • akdsfl.com
  • sex.com
  • rarecats.com

As a rational economic actor, you would likely choose sex.com, not because it is linked to a pre-existing website, but because it refers to a shared kind of meaning that happens to be very monetizable on the internet. In the list above, Sex.com is the schelling point, the ‘solution people choose by default in the absence of communication’.

Unfortunately you can’t buy sex.com. .com domains can only be owned by a single entity at once. But what if there were 2100 (highly divisible) units for each domain. What could you do with them? This is the question that the next iteration of 2100 will attempt to answer.

Mechanism

In the updated version of 2100, each string has 2100 units. There are 2100 $sex, 2100 $rarecats, 2100 $donaldtrump and 2100 tokens that correspond with your Twitter handle, Instagram name, personal domain, favorite hockey team, or any other economically useful (or useless) topic you can think of. You can create these tokens on the fly as needed for any string so long as the string hasn't already been registered.

One (1) of each string token will be auctioned daily. Auctions consist of a pool of funds which proportionally split the token supply that is distributed daily.

You can start an auction for a new string, or participate in an existing auction for an existing string. Auctions start at 1 wei for 1 token and last 24 hours. If nobody else competes with you for the token supply in your auction, you will receive 100% of the supply. If others send in funds during the auction, the supply is split in proportion to funds sent.

The 2100 Alpha can be built as a single app using this system, with each new app unlocking new kinds of utility for different classes of scarce strings.

  2. Visit beta.2100.co to create a string token or participate in an auction
