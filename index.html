2100

Introduction

In 2100, participants use virtual mining to mint tokens that are linked to Twitter usernames. This process creates a unique currency per Twitter account.

Each account is represented as an ERC-20 token and has a fixed supply of 2,100. Collectively, the top 10,000 Twitter accounts create a supply of 21 million tokens: 10,000 non-fungible groups of 2100.

The owner of a username on Twitter is entitled to 10% of their token's supply (210 tokens). A public marketplace of 'minters' compete to mint and own the remaining 1890 tokens.

To mint 2100 tokens, participants 'stake' the DAI stablecoin on the Twitter username of their choice. Forgoing interest on their staked DAI entitles them to a proportional amount of the username tokens as they are minted.

Motivation

We speculate that 2100 tokens have multiple use-cases and create natural incentives for account owners to create utility for their token.

Prior Art

In a May 2019 interview with @joerogan, @jack recounted that Twitter was not invented but ‘discovered’. With this statement, he was referring to the post-internet inevitability of a Twitter-like protocol.

We theorize that socially-linked commodities are a natural outcome of the democratization of financial engineering, and that 2100 or a similar protocol is also awaiting 'discovery'.

As evidence of this, we cite three previous (though incomplete) implementations of this idea:

The Hollywood Stock Exchange (HSX)

Created in 1999 by @maxkeiser and Michael R. Burns, the Hollywood Stock Exchange, or HSX, is a web-based, multiplayer game in which players use simulated money to buy and sell simulated shares of actors, directors, upcoming films, and film-related options.

https://en.wikipedia.org/wiki/Hollywood_Stock_Exchange

Stolen!

Created in November 2015 by @blader, Stolen! was a fast paced mobile game that quickly gained popularity. The goal of the game was to steal Twitter accounts from other users using in-game currency S-bucks which was earned by holding popular accounts.

https://en.wikipedia.org/wiki/Stolen!

Augur Twitter Markets

Augur is a decentralized prediction market platform built on the Ethereum blockchain that was founded in 2014. In 2019, prediction markets betting on the follower count of Twitter celebrities become popular trading venues.

https://en.wikipedia.org/wiki/Augur_(software)

Towards long-lived, platform independent social commodities

Both Stolen and HSX achieved finite success, but suffered from the problem that their commodity had no value outside of a single app. They both had hardened game mechanics that could not transcend their niche gaming use cases. Stolen also suffered from censorship when Twitter withdrew API access.

Augur Twitter markets have given us the closest approximation of a censorship resistant, platform-independent commodity, however Augur markets predicting Twitter followers exist only for a specific window of time and for a specific range of Twitter followers. Further, follower count can be sybil-attacked and is not always correlated with an increase in popularity or utility.

2100 proposes a mechanism to create long-lived, censorship resistant, and platform independent, identity-linked commodities. If the architecture of the protocol is correct, we hope that it will inspire people to build applications like HSX and Stolen on 'Layer 2' of the protocol.

Schelling point

A casual criticism of Bitcoin is that it ‘isn’t backed by anything’. Those who have studied it more thoroughly understand that Bitcoin is backed by social consensus, and that it is the current ‘Schelling point’ for digital scarcity.

Similarly, there is no intrinsic relationship between 2100 tokens and Twitter accounts. It is a relationship that emerges with the correct functioning of the 2100 protocol and the emergence of token utility that are built on the protocol.

Protocol

On Centralization

While our long term goal is to completely decentralize the 2100 protocol, the costs of fully decentralized staking on the ‘long tail’ of 10,000 Twitter account names are currently too prohibitive. For 2100 to succeed, a stakers ‘cost of production’ for each token must be very low.

For this reason, 2100 will start out with a centralized staking platform that keeps track of user balances and generates tokens “off-chain”. Users can at any time redeem tokens on-chain using a message signed by Citizen Hex’s private keys.

Creating a Token Contract

2100 token contracts are created when Citizen Hex approves that a string of lowercase characters is an existing Twitter username.

Anyone in the world can send a tweet specifying which Twitter username should be added to 2100:

"Add @SnoopDogg to #2100

Citizen Hex will verify that “snoopdogg” is an existing Twitter username, that a token with the name “snoopdogg” has not yet been created, and that “snoopdogg” isn’t on the username blacklist (See “Owner Opt-Out and Blacklist”). Citizen Hex will use its private keys to sign an authorization message.

Anyone can then broadcast this authorization message to the Ethereum blockchain to create the account token contract.

When the token contract is created, 0.0021 tokens will be minted to the user who submitted the transaction as a reward for creating the contract.

Generating Tokens off-chain

After creation, tokens on 2100 username contracts are generated each Ethereum block by Citizen Hex. This generation of tokens takes place in a database owned by Citizen Hex and tokens are not minted on-chain until a user requests settlement. Every block will generate 0.00021 username tokens until the maximum supply of 2,100 is reached.

Users deposit DAI (a token pegged in value to the US Dollar) into a non-custodial Ethereum contract. Citizen Hex never has possession of these funds and users can deposit or withdraw DAI at any time.

When a user deposits DAI, Citizen Hex detects this amount and credits the user with the equivalent staking allocation in a database owned by Citizen Hex. When a user withdraws DAI, Citizen Hex removes the staking allocation in its database.

Users can log on to the 2100 app to see their staking allocation. They then choose how much of their allocation they want to stake on particular user name accounts.

For every Ethereum block detected by Citizen Hex, 0.00021 tokens are generated in a Citizen Hex database for each username account. 90% of these tokens are divided pro-rata to users who have staked on these accounts. 10% of these tokens are reserved for the Twitter account owner as an ‘owners reward’ ((see “Owners Reward”).

For example, Alice has chosen to stake 1 DAI on $SnoopDogg and Bob has staked 2 DAI. For each block, Alice will be credited (1/3*0.00021*90%) 0.000063 $SnoopDogg, Bob will be credited (2/3*0.00021*90%) 0.000126 $SnoopDogg, and @SnoopDogg will be credited (0.00021*10%) 0.000021 $SnoopDogg.

Minting Tokens on-chain

At any time a user can choose to take their off-chain tokens and mint them on-chain. Once on-chain these assets are fully interoperable with other tokens on the Ethereum network.

A user clicks a button in the 2100 app and the app signs an authorization message for the 2100 smart contract to mint the tokens they are owed. When the user broadcasts this authorization message on-chain, the 2100 smart contract mints the tokens.

Following the example above, Alice wants to redeem 0.000063 $SnoopDogg from the centralized app and put it on the decentralized Ethereum blockchain. She clicks “Redeem” in the app and she receives an authorization message. She broadcasts this message to the Ethereum network, sending it to the 2100 smart contract, which instructs the “snoopdogg” token contract to credit her balance with 0.000063.

The 2100 smart contract enforces that the authorization message cannot be used more than once and that the message is signed by Citizen Hex. When a token is minted on-chain it is now fully in possession of the user and can be used however she wants.

Owners Reward

Owners of a Twitter username will be credited 10% of the minted token issuance on the contract that shares their username. Until an owner's address is known, rewards will be issued to the zero address (0x0000000000000000000000000000000000000000).

Owners of a username on the Twitter platform can change the issuance address and claim any existing tokens to their own Ethereum address by tweeting out a message specifying their Ethereum address.

Citizen Hex will monitor Twitter for this tweet and sign an authorization message that claims any current and ongoing owner’s reward tokens to a specific address. This tweet mechanism that only the owner of a Twitter account can claim their owner’s rewards.

When this message is broadcast on-chain, the 2100 smart contract will verify its authenticity and store the owner’s address. The 2100 centralized app will detect this change and credit the owner’s Ethereum address with any outstanding tokens, and will issue new tokens to this address.

An owner can then mint their tokens on-chain like any other user of the application.

Username changes

Twitter allows username owners to change their username. This changes the url of the account profile and how it is mentioned in tweets, but keeps all other existing data such as followers, likes, etc.

In order to keep the social consensus that 2100 tokens use Twitter usernames as reference, a 2100 username token must also be able to alter its symbol to match a Twitter username change.

When a Twitter account is renamed, anyone in the world can send a tweet specifying which username changed:

Citizen Hex will verify that this happened and sign a message authorizing a name change on the $andrewyangvfa token contract.

Anyone can broadcast this message on-chain to change the token’s name and symbol. If an existing 2100 token contract exists with the new symbol (andrewyang), it will be changed to the old symbol (andrewyangvfa).

The address of the 2100 token contract does not change at the end of this process and thereby closely mirrors the Twitter platforms username change feature.

This mechanism deters stakers from moving funds out of the old username token contract and into the new username token contract at the moment the Twitter account changes its username.

The intent of the move is to be the first stakers in a smaller pool on the official username contract and to devalue holders of the old username contract token. However, after a user or Citizen Hex detects the change, this discrepancy will be fixed and these opportunistic stakers will instead be mining unofficial and thus devalued tokens.

Owner Opt-Out and Blacklist

Should the owner of a Twitter username not want their username to be used in the 2100 app they can tweet a message:

Citizen Hex will verify that this tweet was sent by the account that wants to opt-out.

If there is no account token with that username, Citizen Hex will broadcast a message on-chain that will add this username to a blacklist. The token contract for this name will never be able to be created once a name is on the blacklist.

If there is a token contract that has its symbol set to the same string as the user’s account, Citizen Hex will broadcast a message that both adds the name to the blacklist and also scrambles the symbol and name of the token contract by hashing the name with the current block number. For example, the token with the symbol “snoopdogg” will have its symbol changed to “6455ffc29962d86b8b08c96b3db27ca0958a1bf1768ccecfde467d3b2bb0b782”

The 2100 centralized app will detect this change and immediately halt staking on this account. Users who have generated tokens on this account can still mint them on-chain, but no further tokens will be generated off-chain or minted.
